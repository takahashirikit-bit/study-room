# スクリプタブルレンダーパイプライン（Scriptable Render Pipeline）

理解するためには以下の用語理解が必要

- <font color="Red">ドローコール </font><br>
描画命令のこと
CPUからGPUに対して複数回描画命令を行う
頂点情報とテクスチャなどのリソースデータをグラフィックメモリに送信
- <font color="Red">レンダーパイプライン（Render Pipeline）</font> <br>
リソースデータとドローコールの発行を制御するメカニズムのこと

レンダーパイプラインには複数の方式が存在する。
方式により、性能や速度に違いがある。

ドローコールが何度も行われて画面の描画がされるため、レンダーパイプラインによってUX体験が大きく変化しそう。

ドローコールを発行する際は、CPUとGPUが同期する必要があり、処理負荷が高い。
同期はそりゃそうか。CPU→GPUに指示するから。

レンダーパイプラインの選定がゲームのクオリティを決定するのに重要な要素である。

しかし、これまでのUnityはユーザーがレンダーパイプラインを選択することはできなかった。

「Unity 組み込みレンダーパイプライン」を使用していた。
これは非常に高性能で幅広いプラットフォームそれぞれにハイクオリティで対応していた。
が、デメリットが多く廃止された。

デメリット
・ユーザーがレンダーパイプラインをカスタマイズできない
→ C++で書かれていてUnityのコア実装に一体化しているため
・実装が巨大で複雑
→万能がゆえ複雑でメンテコストが高い
・新しい技術の取り込みが難しい
→コア一体化しているため気軽に更新できない

そこで開発されたのがSRP。
要は「**プログラミング可能なレンダーパイプライン**」

SRPは、レンダーパイプラインの各機能をモジュール単位で分解し、それぞれをC#経由で呼び出すことができる。
SRPはUnityが提供するC#ライブラリのパッケージ管理フレームワークである「Package Manager」の仕組みを介して提供される。

Package Manager経由で提供される機能はUnityとは別で管理されるためUnityのバージョンに依存しない。
必要なパッケージライブラリを追加したりバージョン更新したりが自由にできる。

Unityは低レイヤーであるSRCCoreをベースにした2種類の公式レンダーパイプラインを提供している。

・UniversalRP(Universal Render Pipeline）  
→以前のパイプラインの後継
・HDRP（High Definition Render Pipeline）  
→PS4などのコンシューマ機やPCなどハイエンド環境での利用を想定。高い精度で表現力がある分GPUに要求する性能が高い

<mark>まとめ </mark>  
・レンダーパイプラインが描画の速度や見た目を決める大きな要素
・これまでのレンダーパイプラインはUnityのコア実装と一体化していた
・SRPが発明され自由にカスタマイズできるようになった
・SRPとUnityは独立しており互いに依存していない
